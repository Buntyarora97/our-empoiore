# Our Empire — Deployment Guide

## Architecture Overview

```
[Player App (Expo/APK)]  →  [API Server on Render]  →  [Neon PostgreSQL]
[Admin Panel (Hostinger)]  →  [API Server on Render]
```

---

## 1. Neon Database Setup

1. Go to [neon.tech](https://neon.tech) → Sign up / Login
2. Create a new project → Name: `our-empire`
3. Select region: **Singapore** (closest to India)
4. Copy the **Connection String** (looks like: `postgresql://user:pass@ep-xxx.ap-southeast-1.aws.neon.tech/neondb?sslmode=require`)

### Run DB Migration

After cloning this repo locally, run:

```bash
npm install -g pnpm
pnpm install
export DATABASE_URL="your_neon_connection_string_here"
pnpm --filter @workspace/db run push
```

This creates all tables: users, markets, bets, transactions, results, admins, settings, notifications, otps.

---

## 2. Render (API Server) Deployment

1. Go to [render.com](https://render.com) → Sign up with GitHub
2. Click **New** → **Web Service**
3. Connect your GitHub repo: `Buntyarora97/our-empoiore`
4. Render will auto-detect `render.yaml` — click **Apply**
5. Set Environment Variables:
   - `DATABASE_URL` → your Neon connection string
   - `ADMIN_DEFAULT_PASSWORD` → your admin password (default: `admin123`)
   - `JWT_SECRET` → auto-generated by Render
6. Click **Deploy**

Your API will be live at: `https://our-empire-api.onrender.com`

> **Note:** Free tier sleeps after 15 min inactivity. Upgrade to Starter ($7/mo) for always-on.

## 3. Admin Panel (Hostinger) Deployment

The admin panel is a **static website** — upload the built files to Hostinger.

### Build the admin panel:
```bash
PORT=20130 BASE_PATH=/ pnpm --filter @workspace/admin-panel run build
```
Built files will be in: `artifacts/admin-panel/dist/public/`

### Upload to Hostinger:
1. Login to Hostinger → **File Manager**
2. Go to `public_html/` folder
3. Upload ALL files from `artifacts/admin-panel/dist/public/` 
4. **Important:** Upload `_redirects` file (included) for React Router to work

### Update API URL in admin panel:
Before building, set the API URL in `artifacts/admin-panel/src/main.tsx`:
```
setBaseUrl("https://our-empire-api.onrender.com")
```

### CORS Setup:
In `artifacts/api-server/src/app.ts`, update CORS to allow your Hostinger domain:
```typescript
app.use(cors({ origin: ["https://yourdomain.com", "http://localhost:20130"] }));
```

## 4. Player App (APK) Build

### For Android APK:
```bash
# Install EAS CLI
npm install -g eas-cli

# Login to Expo account
eas login

# Build APK
cd artifacts/player-app
eas build --platform android --profile preview
```

### Update API URL in player app:
In `artifacts/player-app/app/_layout.tsx`, the URL is set via:
```
EXPO_PUBLIC_DOMAIN=your-render-domain.onrender.com
```

Add to `artifacts/player-app/.env`:
```
EXPO_PUBLIC_DOMAIN=our-empire-api.onrender.com
```

| Variable | Value |
|----------|-------|
| `EXPO_PUBLIC_DOMAIN` | `our-empire-api.onrender.com` |

## 6. Admin Login

- URL: `https://yourdomain.com/` (Hostinger)
- Username: `admin`
- Password: whatever you set in `ADMIN_DEFAULT_PASSWORD` (default: `admin123`)

## Tech Stack

- **API Server:** Node.js + Express 5 → Render (free tier)
- **Database:** PostgreSQL → Neon (free tier, 0.5GB)
- **Admin Panel:** React + Vite → Hostinger (static files)
- **Player App:** Expo React Native → APK download / Expo Go
